After the user enters their Emp Id and Password and clicks Login, the app checks if the credentials match the expected values. If they do, instead of storing the actual password or Emp Id, the app creates a simple token or flag, like `"accessToken": "true"`, and saves it in localStorage. This token represents that the user is logged in and allows the app to remember the login even after a page refresh or browser restart. The `isLoggedin` state is updated immediately so the UI responds to the login, and the user is navigated to the Dashboard. In the PrivateRoute, the app checks if this token exists in localStorage; if it does, the Dashboard is shown, and if not, the user is redirected to the Login page. This way, the login state persists safely without storing sensitive information like the password.
